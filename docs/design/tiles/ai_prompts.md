## 8. Post-Processing Instructions

After generating each sprite:

1. **Resize**: Scale to exactly 32x32 using nearest neighbor (no blur)
2. **Palette**: Remap colors to official 32-color palette
3. **Outline**: Add/verify 1px `#0a0a12` outline on characters
4. **Transparency**: Ensure background is true alpha, not color
5. **Check**: Verify silhouette is readable and distinct
6. **Compare**: Check consistency with other sprites

---

## 9. Batch Generation Strategy

1. Generate **4-8 variants** of each sprite
2. Select **best silhouette** from batch
3. Apply **palette normalization** to winner
4. **Touch up** in pixel editor (Aseprite) if needed
5. **Test in tileset sheet** for consistency
6. **Regenerate** any that don't fit

---

## 10. Tool-Specific Notes

### DALL-E 3
- Add "pixel art" prominently at start
- Specify exact dimensions: "32x32 pixels"
- May need multiple attempts for clean results

### Midjourney
- Use `--ar 1:1` for square output
- Add `--style raw` for less stylization
- Upscale then downscale to 32x32

### Stable Diffusion
- Use pixel art LoRA model
- Set output to 32x32 or 64x64 (then scale)
- Negative prompt: "blurry, anti-aliased, smooth"

### Aseprite (Cleanup)
- Import AI output
- Canvas resize to 32x32
- Index to 32-color palette
- Manual pixel cleanup
- Export PNG with alpha
