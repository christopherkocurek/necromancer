## Object.txt File Format

```
N: serial_number : & object_name~
G: symbol : color
I: tval : sval : pval
W: depth : rarity : weight : cost
P: plus_to_hit : damage_dice : plus_to_evasion : protection_dice
A: depth/rarity : depth/rarity : etc
F: flag | flag | etc
D: description
```

### Key Fields for Renaming:
- N: Contains the item name (what we're renaming)
- I: tval determines item category
- W: depth determines layer appropriateness
- D: description/flavor text

## Design Principles
1. tolkien_grounded: Names should evoke Middle-earth, not generic fantasy
2. layer_appropriate: Items found deeper should sound darker/more corrupted
3. functional_clarity: Names must still communicate what the item does
4. no_first_age: Avoid Silmarillion-era references. Third Age only.
5. no_implementation: This PRD produces design docs ONLY.
